Feedback page

Technical Description

HTML and CSS are used to create the **Feedback Page**, a form that enables users to put forward their opinions, comments, and improvements to the site. The primary HTML tags and CSS properties employed are:

    HTML Structure:
    • <form>:This web element includes all the input fields and bank details needed to be completed in a user-friendly way by the customer.

    • <fieldset>: These are groups related to the form elements. It is used to enhance the structure and readability of the form.

    • <input> and <textarea>: Here the text, email, and feedback adding are the basic ones.

    • <label>: This feature helps to differentiate each input field. Therefore, it becomes clearer and more understandable for the user.

    • <button>: Users be able to submit the form.
    CSS Styling:
    • <body>: It uses a bright colored wallpaper (`#c9d6fa`) so the user can enjoy the visual experience..

    • <content>: Lets the form be more readable and also allows for more efficient spacing and padding to be applied.

    • <input[type="text"], input[type="email"]>: he styled text field has padding, border-radius, and width properties to create an intuitive and accessible input field..

    • <rating>: It uses the **grid layout** for a thought-out rating selection box.

    • <button: It prompts a visual feedback as the button color shifts when a user moves the mouse cursor over, thus improving UX.

    • <header>: It features a **hidden navigation bar** using JavaScript to define a boundary and a threshold to make navigation bar appear with the toggle scroll button. It has a background color of white. A **scroll-based** method with **CSS only** is used to change colors in a flashy manner on the bar.

Accessibility

    Accessibility has been considered in the following ways:

    • <label> : This is so because this means that all of the form fields shall be provided with alternate text. Screen readers can now convert the text for the visually impaired.

    • Contrast and Readability: A clean layout and high-contrast colors will enhance the visibility of text.

    • Keyboard Navigation: Users can navigate between the form fields using the `Tab` key or other shortcut keys..

    • Required Fields: These are the fields which are indicated by the user input code, for instance by higly respectcted designers as a go-to thing for their design.

    • Responsive Design: The form scales to various screen sizes using relative widths and margins.

Team page

Technical Description

    HTML Structure:
    • <header> : A logo and navigation menu that is fixed to the top of the page are contained within this section. The background color of the header changes when scrolling of the user is done, thus making it more visually appealing.

    • <section class="team">: This is the main section which has a full-page layout with a radial gradient background and the images, names, and roles of the team members are included in it.

    • <div class="box">: Every team member is given in the form of a "box" element. The application of the hover effect on the boxes, which reveals more details about the tasks the participants worked on, is shown using CSS.

    • <footer>: Links for each editor, their name, and a notice that the page validates and is accessible to everyone are included.

    CSS Styling:
    • Body and Layout: The fonts used are Poppins and Montserrat. The background is set to a dark, water-like color (#000000), creating a visual impression of the ocean or underwater theme. Flexbox and grid systems are used for page layout, so that content is responsive.

    • Header Styling: Although the navigation bar is initially transparent, it changes its color during user scrolling, thanks to JavaScript that adds the scrolled class to the header.

    • Team Member Boxes: The design of each box gets a dark finishing with rounded corners. An animation is set for each box, which will show it by moving up and displaying extra task information in an animated way.

    • Footer: The footer shares the same color with the header, thus keeping the design intact. It also offers links to each page editor for accessibility

Accessibility

  • Responsive Design: This page implements flexible grid layouts and relative units to ensure an optimal viewing experience for all screen sizes.

  • Hover and Focus States: The links in the navigation and team member boxes have their unique hover states for an easier and more efficient page navigation.

  • Contrast: The design indeed offers proper contrast between the bigger the text and background, thus enhancing readability.
  • Navigation: Through the new design, users will be able to interact with the team members and navigate the rest of the site by using a fixed, easily accessible header.

Interactive Features

  • Hover Effects: For instance, when the users point to a team member’s image, a more comprehensive explanation about their role and contributions is displayed, which creates a dynamic and intuitively interacting interface. This is enabled by CSS transitions and hiding/showing content.
  • Fixed Navigation: The header is always fixed at the top of the page to easily view the different sections using the anchor link without needing to go to the top.

Challenges and Lessons Learned

Web development had a plot to tell, and it was not a fairytale, for there were some struggles on the way, which worked towards developing technical skills and problem-solving.” One of the main issues was the user experience was not smooth across different devices. The first step was to make the design more suitable for small screens, because of the missing optimization which in turn affected navigation and legibility. The following were the main accomplishments realized: the use of FlexBox and CSS Grid along with the philosophy of using 'media queries' of responding to lay-out resizing by dynamically changing the UI. On the other hand, accessibility was another challenge, as the website did not contain the necessary alt attributes and ARIA labels for the images and interactive elements, respectively. The correction of these issues resulted in the website being more accessible to the group of people who have various types of disabilities. These difficulties brought accessibility and responsiveness into sharp focus, demonstrating the necessity of integrating these elements from the beginning, otherwise, they will turn into bulky makeovers.

Compliance

The website is designed to be accessible by people with disabilities ensuring they would have inclusive experience. It is guiding its users across different devices via the mobile-friendly responsive layout. The user’s data has the necessary protection and other GDPR standards are ensured. The content is in line with Janet’s guiding rules, which are eco-responsibility, digital citizenship, and educational purpose
