Profile Page

Technical Description

The profile page is implemented with semantic HTML tags including <header>, <nav>, <section>, and <div> to create a logical and accessible structure. Flexbox is used to center content and manage layout in a responsive manner. Inline CSS styles complement an external stylesheet for specific profile card elements (such as border-radius, box-shadow, and progress bars) to enhance the visual hierarchy and user experience.

Accessibility

Accessibility has been addressed by including descriptive alt attributes for images, ensuring sufficient color contrast, and using semantic tags to assist screen readers. The interactive elements, like progress bars and navigation links, are designed to be keyboard navigable.

Sitemap Page

Technical Description

The sitemap page uses an <svg> element to display a visual map of the website’s navigation. Rectangular nodes ( <rect>) represent individual pages and are linked with <line> elements. Text labels are added with the <text> tag. The SVG is styled with CSS to match the overall website design and includes hover effects along with keyboard navigation enhancements via tabindex and JavaScript key event listeners.

Accessibility

Accessibility is improved by assigning a tabindex to each node for keyboard navigation and ensuring that interactive SVG elements respond to both click and keypress events. The use of descriptive <title> tags within the SVG further aids screen readers.

Content Page

Technical Description

The content page (focused on Marine Pollution) is structured using a hero section with a full-width background image and overlaid text with text-shadow for improved readability. Sections are organized with <section> tags, and headings (<h1> and <h2>) structure the content for both users and search engines. CSS properties such as background, padding, margin, and border-radius are employed to create a modern, responsive design that adapts across devices.

Accessibility

Accessibility improvements include the use of semantic HTML elements, descriptive alt text for all images, and careful color contrast choices. The layout supports keyboard navigation through clear focus indicators and logical heading order.

Challenges and Lessons Learned

One of the major challenges was ensuring cross-browser consistency while integrating both inline and external CSS. In addition, making interactive elements such as the SVG nodes fully accessible required adding extra JavaScript to support keyboard events. These challenges taught me the importance of modular code and thorough testing on various devices and browsers. It also emphasized the need for consistent design practices and early accessibility testing.

Compliance

All pages are developed in compliance with Janet regulations for web page publication. Semantic HTML, proper use of heading tags, and valid CSS ensure that each page is both accessible and search engine friendly. Furthermore, each page was validated with the W3C validator to ensure that the code meets modern web standards and accessibility guidelines.
